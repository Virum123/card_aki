### 25) 추천 로직 점수화 전환 (중요)
- Y/N 하드필터 중심 구조를 완화하고, 질문 응답별 점수 합산 방식으로 전환.
- 하드필터는 `실적/연회비`만 유지하고, 생활/선호 질문은 혜택 강도(0~1) 기반 점수로 반영.
- 질문 완료 후 카드별 총점을 `100점 만점`으로 환산해 정렬.
- 결과 노출은 상위 `100개`로 제한(`TOP_N_RESULTS=100`).
- 편의점/쇼핑/구독 등은 "해당 혜택 강도"에 따라 0~100 사이 연속 점수로 계산.

### 26) 질문 란 모드 분리 + 세부탐색 5점척도
- 질문 란에 `간편 탐색` / `세부 탐색` 모드 선택 추가.
- 간편 탐색: 기존 Y/N 하드필터(실적/연회비 + 생활질문) 유지.
- 세부 탐색: Y/N 질문을 5점 척도(1~5) 입력으로 전환.
- 세부탐색은 하드필터를 실적/연회비로 제한하고, 나머지는 가중 점수 반영.

### 28) 점수 모델 고도화

- 혜택 강도는 카드풀 내 상대 강도(0~1)로 정규화해 반영.

- 세부 탐색의 5점 척도를 슬라이더에서 대형 클릭 박스(1~5 버튼) 입력으로 변경.

### 30) 혜택 하이라이트 문구 구체화
- 결과 카드의 혜택 하이라이트를 단순 충족률(예: 67%) 표기에서, 요약문 기반 혜택 문구(예: `간편결제 시 최대 n% 적립/할인`) 중심으로 변경.
- 요청 혜택 충족도가 높을수록 색상/볼드 강조를 유지.

### 31) 점수/입력/결과 UX 보완
- 세부 탐색 5점 입력을 박스형 선택 UI(가로 선택)로 변경하여 클릭 하이라이트 안정화.
- 점수 계산식을 세부 탐색 기준 `필요 정도(1~5 정규화) * 혜택 강도` 형태로 조정.
- 결과 혜택 문구를 `% 충족` 중심에서 `n% 적립/할인` 중심 문장으로 전환.
- 카드 상세 링크를 카드고릴라 링크 대신 카드사 공식 페이지 링크로 변경.
- 세부 탐색에서 상충 강한 요구(예: 5점-5점) 발생 시 우선순위 추가 질문(2지선다)을 노출.
- 요청 혜택 충족도는 "선택 없음" 대신 사용자 선호 점수가 높은 상위 3개 혜택을 하이라이트해 표시.

### 32) 입력/점수/이미지 신뢰성 보완
