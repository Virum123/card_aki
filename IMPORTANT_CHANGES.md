# Important Changes Log

## 2026-03-03

### 1) 전체 구조 전환
- 기존 FastAPI + React 구조를 제거하고 Streamlit 단일 앱 구조로 전환.
- 프로젝트 목적을 "카드 DB 수집 + 빠른 검색 프로토타입"으로 재정의.

### 2) 크롤러 및 DB 생성 파이프라인 추가
- `crawler/build_card_db.py` 추가.
- 수집 대상: `https://card-search.naver.com/list`
- 저장소: `data/cards.db` (SQLite)
- 보조 검증 파일: `data/cards_snapshot.json`

### 3) Streamlit MVP 화면 추가
- `app/streamlit_app.py` 추가.
- 기능: 카드명 검색, 카드사 필터, 목록 조회.

### 4) 전체 카드 수집 방식 전환 (중요)
- HTML 파싱 방식에서 GraphQL `smartSearch` 페이지네이션 방식으로 전환.
- `pageNo`를 증가시키며 더보기 동작을 재현해 전체 카드(기준 361건) 수집 가능하도록 변경.
- 테이블 스키마에 `card_ad_id`를 추가하고, 실행 시 테이블 재생성으로 스키마 충돌을 방지.

### 5) 연회비/혜택 필드 확장
- `cards` 테이블에 `summary`, `domestic_annual_fee`, `foreign_annual_fee`, `family_annual_fee`, `benefit_tags` 컬럼 추가.
- Streamlit 화면에서 혜택 키워드 검색, 국내 연회비 범위 필터를 지원하도록 개선.

### 6) 질문형 페이지 구조 전환
- 카드 검색형 UI에서 질문(설문) 기반 추천 UI로 전환.
- 질문 정의를 `app/question_bank.py`로 분리해, 질문 문구/옵션을 코드에서 별도 관리하도록 변경.
- 설문 제출 시 `filter` 규칙(`contains`, `equals`, `lte`, `gte`, `between`)으로 카드 DB를 필터링하도록 구현.

### 7) 브랜드/기획 문구 반영
- 서비스 타이틀을 `카드의요정`으로 변경.
- 기획 의도, 유사 서비스 비교, 서비스 지향점 문구를 README 및 Streamlit 화면 상단에 반영.
- 제공된 13개 질문을 설문 목록으로 등록.

### 8) 단계형 문답 UI 전환
- 질문 전체 노출 방식에서 한 번에 한 질문만 보여주는 단계형 UI로 변경.
- 안내원 톤의 질문 메시지, 진행률 표시, 이전/다음/결과보기/처음부터 버튼을 추가.
- 마지막 질문에서 `결과 보기`를 눌러야 추천 결과가 표시되도록 흐름을 분리.

### 9) 실시간 후보 수 축소 표시
- 각 질문 답변 직후 현재 조건을 만족하는 카드 수를 실시간(`현재 조건 충족 카드 수`)으로 표시.
- 안내원 질문을 채팅 버블(아바타 포함) 형태로 변경해 대화형 UX를 강화.

### 10) 지표/노션 계획 문서 분리
- 질문 기반 카드 필터링 품질 관리를 위한 지표 문서 추가: `docs/DB_FILTER_METRICS.md`
- 노션 연동은 구현 보류 상태로 계획 문서만 분리: `docs/NOTION_INTEGRATION_PLAN.md`

### 11) 질문 규칙 V2 반영
- 연회비 질문을 `3만원/10만원 +/-` + `직접입력(±1만원)` 구조로 변경.
- 해외여행/직구 질문을 Y/N으로 통일하고, 해외결제 혜택 보유 카드 수 지표를 화면에 추가.
- 마일리지 관심 질문 후 `대한항공/그 외` 후속 질문 분기 추가.
- 생활영역(카페/외식/쇼핑/편의점/통신/OTT/간편결제/차/교통)을 혜택 키워드 기반 필터로 일괄 정리.
- 인터넷전문은행 질문은 필터가 아닌 카드사 가점 정렬 규칙으로 적용.

### 12) 필터 정밀도 튜닝 (1단계)
- 단순 전체 텍스트 검색에서 `benefit_tags` 우선 매칭 방식으로 개선.
- 태그 alias 매핑(`TAG_ALIAS_MAP`)을 추가해 카테고리별 오탐을 줄임.
- 태그 매칭 실패 시에만 fallback 키워드 검색을 수행하도록 변경.

### 13) 최소 실적 기준 필터 도입
- 카드 상세 페이지의 `baseRecord.valueFrom`를 추출해 `min_spend_required_krw` 컬럼을 DB에 추가.
- 월 사용금액/희망 실적 질문을 최소 실적 기준 필터에 연결.
- 추천 결과 테이블에 `min_spend_required_krw`(최소 실적) 컬럼 노출.

### 14) 연회비 질문 범위 슬라이더 전환
- 연회비 질문을 규칙 선택형에서 범위 슬라이더형으로 변경.
- 범위: `0원 ~ 10만원+` (`110000` 끝점은 상한 없는 `10만원+` 의미).

### 15) 추천 로직 점수형 전환
- 필터 중심 구조에서 `하드 필터(실적/연회비) + 매치율 점수` 구조로 전환.
- 해외/마일리지/생활혜택/간편결제/교통/자동차/주거래은행은 매치 점수에 반영.
- `대한항공/그 외`는 하드 필터가 아닌 소프트 가점(순위)으로 처리.
